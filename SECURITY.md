# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of this project seriously. If you believe you have found a security vulnerability, please report it to us by following these steps:

1.  **Do not disclose it publicly**. Please avoid opening a public GitHub issue or mentioning it on social media.
2.  **Send an email**. Reach out to [klarence.nevado@gmail.com](mailto:klarence.nevado@gmail.com) with the following details:
    -   Type of vulnerability (e.g., SQL injection, sensitive data exposure).
    -   Steps to reproduce the issue.
    -   Potential impact of the vulnerability.
3.  **Wait for a response**. We aim to acknowledge receipt within 48 hours and provide a timeline for a fix.

## Our Commitment

We appreciate your help in keeping this project secure. We will:
-   Acknowledge and validate your report.
-   Provide updates on the progress of addressing the issue.
-   Publicly credit your contribution to the security of the project (if desired) once a fix is released.

Thank you for your cooperation!
